# Bush Riding — repo guide for Claude

This repo is the whole Bush Riding world: the Shopify theme (repo root), the
routes map site (`map/`), its two Cloudflare Workers (`worker/`,
`diary-worker/`), and the ALP1NE design study (`alp1ne/`). Read this file
before making changes; the Brand voice section governs ALL copy written
anywhere in the project.

## Brand positioning (source: "Bush Riding — Brand Positioning", March 2026)

**Positioning statement.** For gravel cyclists aged 20–35 who value
aesthetics, community, and authentic outdoor experiences over performance
metrics, Bush Riding is the fashion-led technical apparel brand that
prioritises peaceful exploration and genuine connection — designed for riders
who care more about the detour than the podium. Unlike performance-obsessed
cycling brands, Bush Riding celebrates the journey, community, and style.

**Audience — "The Conscious Explorer".** 20–35, urban-based but seeks
gravel/nature, mid-tier premium disposable income, brand-conscious. Values
identity expression, authenticity, community, aesthetics. Into film
photography, storytelling, exploration. Tired of boring outdoor brands;
doesn't want to look like "dad in Lycra". Rides 100 km but doesn't obsess
over pace; organises café stops; wants to look good on AND off the bike.
Buys into culture, not just product function.

**Core promise.** Technical gravel apparel that lets you explore in style,
connect with community, and express your authentic self on every ride.
Proof points: designed in Australia; premium materials at accessible mid-tier
pricing; community-first launch (Strava groups, monthly rides);
photography-led storytelling; gender-neutral.

**Personality.** Playful, authentic, relaxed, community-oriented, culturally
aware. Secondary: innovative (but not loud about it), fashion-conscious,
slightly irreverent.

**Tone of voice.** Effortlessly cool and grounded — a seasoned explorer who
doesn't need to brag. Conversational, like sharing stories around a campfire.
"We speak like humans, not marketing departments."

Rules that shape every sentence:
- **Exploratory & curious** — frame rides as adventures, not workouts;
  celebrate unknown paths and the "bush" mentality.
- **Community-first** — say "we"/"us", not "you". No competitive posturing.
- **Comfortably technical** — explain features through experience ("so you
  can ride longer"), never spec-speak first.
- **Responsibly made** — honest, thoughtful, never preachy. Australian roots
  matter: local design, global perspective.
- **Lead with feeling, support with function.** Journey over destination.
  Authentic over polished. Show, don't sell ("we document, we don't
  advertise"). Inclusive, not exclusive (gender-neutral, accessible pricing).

Voice examples (from the document):
- Not "Our premium gravel jersey features cutting-edge fabric" →
  "Designed for those long days when the road turns to dirt and you're not
  sure where you'll end up — in the best possible way."
- Not "Join our exclusive community." → "Ride with us. Share your stories.
  Let's explore together."
- Not "High-performance cycling apparel." → "Kit that moves with you, built
  for comfort on every surface — because the best rides are the ones where
  you forget what you're wearing."

**Visual identity.** Documentary/film photography over polished campaign
style; Australian landscape; product shown in action with an adventure
storytelling lens; community-focused imagery with a sprinkle of
fashion-forward shots. Minimalist type logo with a vintage feel (old outdoor
guides). Muted colour palette inspired by Australian landscapes. Imagery does
most of the work — text and icon as support. Show, don't tell.

## What lives where

- **Repo root** — the Shopify theme (Dawn-based). The published store theme
  follows `main`; Shopify's GitHub integration syncs connected branches.
- **`theme/alpine` branch** — draft theme wearing the ALP1NE-derived skin
  (sections `alpine-*.liquid`, `assets/alpine.css`/`alpine.js`, tokens
  `--alp-*`). Connected to Shopify as an unpublished theme; preview there.
  Note: its default copy ("ALP1NE™", PST clock, FW26 lines) is placeholder
  from the design study — real copy must follow the Bush Riding voice above.
- **`map/`** — bushridingmap.com… now **map.bushriding.cc**: static MapLibre
  routes map deployed by Cloudflare Pages (output dir `map`), plus Pages
  Functions in `functions/`. SEO route pages under `map/routes/**` are
  GENERATED by `scripts/generate-route-pages.js` — never hand-edit them.
- **`worker/`** — Cloudflare Worker `bush-riding-map-api`
  (map-api.bushriding.cc): community route submissions, moderation, events.
- **`diary-worker/`** — Cloudflare Worker `bush-riding-diary`
  (diary.bushriding.cc, reached same-origin via the `/diary-api` Pages
  proxy): accounts + ride diary. Both Workers deploy via Cloudflare's
  Git-connected builds (root dirs `/worker`, `/diary-worker`).
- **`alp1ne/`** — the original static ALP1NE design study; reference only.

## Branch workflow

`main` = production. `develop` = integration; day-to-day work lands here
first, publish by merging `develop` → `main`. `feature/*` for experiments;
`theme/alpine` for the ALP1NE theme build. Never force-push shared branches.

## Hard-won rules

1. **Always `git pull --rebase` before editing on a Shopify-connected
   branch.** The Shopify bot commits editor changes back ("Update from
   Shopify…") and has at least once pushed a STALE copy of assets, silently
   reverting recent code. After pulling, check that your recent changes are
   still present in the files you're about to touch.
2. `templates/*.json`, `sections/*-group.json`, `config/settings_data.json`
   are shared with the theme editor — treat them as editor-owned; change
   code-owned files (`sections/*.liquid`, `assets/*`) freely.
3. Theme-editor compatibility: sections must always carry a complete
   `{% schema %}`; the editor re-renders section HTML on every tweak, so any
   JS that reveals/animates content must handle `shopify:section:load` and
   `Shopify.designMode` (see `assets/alpine.js`).
4. Everything in `assets/alpine.css` is prefixed `alp-` to avoid colliding
   with Dawn's `base.css`. Keep it that way.
5. The map site and Workers changed domain from bushridingmap.com to
   map.bushriding.cc / map-api.bushriding.cc — old-domain references only
   belong in `map/_redirects` (301s) and nowhere else.
6. **Theme design changes are scoped per platform.** Mobile and desktop are
   deliberately designed differently. If a request doesn't say whether it
   applies to mobile, desktop/tablet, or both, ASK before building — don't
   assume both.
7. Render theme images with `image_url | image_tag` (never CSS
   background-image) so the focal point set in the Shopify editor is honoured.
